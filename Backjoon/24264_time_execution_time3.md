## 알고리즘 수업 - 알고리즘의 수행 시간 3
### 문제
입력 크기 n이 주어지면 아래 알고리즘의 수행 시간을 출력해보자.
```
MenOfPassion(A[], n) {
    sum <- 0;
    for i <- 1 to n
        for j <- 1 to n
            sum <- sum + A[i] × A[j]; # 코드1
    return sum;
}
```

### 입출력
- 입력 : 첫째 줄에 입력의 크기 n(1 ≤ n ≤ 500,000)이 주어진다.
- 출력 :
  둘째 줄에 코드1의 수행 횟수를 다항식으로 나타내었을 때 최고차항의 차수를 출력.
  (단, 다항식으로 나타낼 수 없거나 최고차항의 차수가 3보다 크면 4를 출력)


### 풀이과정
#### 코드 1은 몇 번 반복할까?
내부 for문에서 j가 1~n만큼 반복하므로 n번, 바깥쪽 for문에서도 n번을 돈다.
즉 n^2만큼 반복하게됨
O(n^2)

#### 코드 1의 수행시간을 다항식으로 나타내면?
f(n) = n^2

#### 최종 코드
```
import java.util.Scanner;

public class Main{
    public static void main(String args[]){
        Scanner sc = new Scanner(System.in);
        long n = sc.nextInt();
        
        System.out.println(n*n);
        System.out.println(2);
    }
}
```

첫번째 n*n을 출력하는 과정에서 틀렸다는 답이 나오는데 제한 크기인 1 ≤ n ≤ 500,000을 넘어서 생긴 문제였다.
n의 범위를 기준으로 n^2을 하면 2,500억까지 나오게되는데 자바의 int자료형은 약 21억까지만 표현이 가능하다.
그렇기 때문에 범위를 초과하는 오버플로우가 발생한 것.
따라서 자료형을 int가 아닌 long으로 받아 해결하였다.

이에 더해 자바는 확장 형 변환 규칙 덕분에 다른 타입끼리 산술 연산을 하면 범위가 더 큰 타입으로 자동 형 변환 후 연산 수행이 된다.
따라서 nextInt()로 반환된 int값을 long타입 변수에 할당해도 n*n이 long끼리 연산으로 수행되어 문제없이 결과값을 얻을 수 있다.
